# 用語ルール

## 「クライアント」＝フロントエンドソフトウェア（人間のユーザーではない）

日本語のソフトウェア文脈における「クライアント」は、クライアントサーバモデルに由来し、サーバに要求を送る側のソフトウェア（ブラウザ上のフロントエンドJS、リクエスト発行元アプリケーション）を指す。人間の利用者（ユーザー）とは明確に区別される。

### CSRFトークンの文脈での含意

「CSRFトークンは正規のクライアントからのリクエストであることの認証」と述べた場合、ここでの「クライアント」＝正規のフロントエンドJSを指す。すなわち「自分のオリジンの正規フロントエンドから発行されたリクエストであること」を意味しており、これはCSRFトークンの役割と一致する。ユーザー識別（リクエストを送った人が誰か）を指しているのではない。

### 適用範囲

- 認証・セキュリティ・CSRF・API設計など、「クライアント」という語が登場するすべての技術的な議論・ドキュメント
