# index-setup スキル

`.claude/docs/` 配下の末端フォルダ（実ファイルが直接置かれるフォルダ）、`.claude/skills/`、`.claude/rules/`、および `.claude/agents/` に、ファイル一覧と役割をまとめた `INDEX.md` を一括作成・更新する。

## 使用シチュエーション

- 新しいプロジェクトにこの `.claude/` 運用（rules/docs/specs/steering 構成）を導入したとき、既存の docs にまだ索引がない場合の初回バックフィル
- 複数ドキュメントをまとめて追加したあとなど、索引が実態とずれた場合の一括立て直し

（1ファイルずつの新規作成時の索引更新は `doc-push-agent` の手順内で自動的に行われる想定。このスキルは「一斉に整備し直す」ための一括処理用。）

## 手順

### Step 1: ルールファイルに索引規約があるか確認する

`.claude/docs/rules/documentation-guide.md`（または同等のドキュメント運用ルールファイル）に「フォルダ索引（INDEX.md）」セクションがなければ追加する。内容:

```markdown
## フォルダ索引（INDEX.md）

`docs/` 配下の末端フォルダ（実ファイルが直接置かれるフォルダ。例: `features/<機能グループ>/`・`rules/`・`tech-spec/`・`migrations/`）、`.claude/skills/`、`.claude/rules/`、および `.claude/agents/` には、フォルダ内のファイル一覧とその役割をまとめた `INDEX.md` を置く。`.claude/skills/`・`.claude/rules/`・`.claude/agents/` はサブフォルダを持たないため、フォルダ自体を1つの末端フォルダとして扱う。

### 目的

サブフォルダの中身は自動的にはコンテキストに読み込まれないため、既存機能に関わる作業を始める前に `find`/`ls` で一覧を取り、さらに中身を推測するために個々のファイルを開く、という手戻りが発生していた。`INDEX.md` を1回読むだけでフォルダの中身と役割を把握できるようにする。

### 形式

| ドキュメント | 役割 |
|---|---|
| <連番を除いたベース名> | <1行で役割> |

同じベース名で連番違いの複数ファイル（例: `user-management-doc-01.md` / `user-management-doc-02.md`）は1行にまとめる。最新版は「最も連番が高いファイルを参照する」という既定で判別できるため、索引側に最新版番号は明記しない。

### 維持管理

`INDEX.md` の作成・更新は `doc-push-agent`（相当のagent）が行う。
```

### Step 2: 対象フォルダを洗い出す

```bash
find .claude/docs -mindepth 1 -type d
```

に加えて `.claude/skills/`・`.claude/rules/`・`.claude/agents/` 自体も対象に含める（いずれもサブフォルダを持たないため、フォルダそのものを1つの末端フォルダとして扱う）。

いずれも、直下に `.md` ファイルを持つフォルダ（末端フォルダ）を対象にする。

### Step 3: 各フォルダのファイルを読み、ベース名でグルーピングする

- ファイル名から連番サフィックス（`-\d+` または `_\d+`）を除いたものをベース名とする
- 例: `user-management-doc-01.md` と `user-management-doc-02.md` → ベース名 `user-management-doc`
- 各ファイルの冒頭（タイトル・`## 概要` セクション）を読み、ベース名ごとに1行の役割要約を書く（連番が複数あっても要約は1行のみ。最新版の判定は「最も連番が高いファイルを参照する」という既定に委ねる）
- 連番サフィックスを持たないファイル（例: `docs/rules/` 配下の各ルールファイル）は、ファイル名をそのままベース名とする

### Step 4: INDEX.md を書く

各フォルダに以下形式で作成・上書きする:

```markdown
# <フォルダ名>/ 索引

| ドキュメント | 役割 |
|---|---|
| <ベース名> | <1行要約> |
```

### Step 5: doc-push-agent（相当のagent）の手順に INDEX.md 更新ステップがあるか確認する

なければ、新規ドキュメント作成時に INDEX.md へ1行追加する手順を追記する（詳細はこのリポジトリの `.claude/agents/doc-push-agent.md` 手順3を参考にする）。

### Step 6: 反映する

このプロジェクトの運用に従いコミット・push する（`doc-push-agent` がある場合はそれ経由、なければ直接）。
